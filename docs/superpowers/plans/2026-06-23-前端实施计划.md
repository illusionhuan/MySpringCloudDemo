# Vue 3 前端实施计划

> **前置条件：** 后端微服务（Gateway Service :8080）已启动并正常运行。

**Goal:** 基于 Vue 3 + Element Plus 构建前端应用，实现用户认证、帖子管理、评论、通知等功能，通过 Gateway 统一调用后端 API。

**Tech Stack:** Vue 3, Element Plus, Vue Router, Axios, Pinia, Vite

## 模块 1：项目初始化

**目标：** 使用 Vite 创建 Vue 3 项目骨架，安装依赖，配置基础结构。

- 初始化 Vite + Vue 3 项目（`frontend/`）
- 安装依赖：`element-plus`, `vue-router`, `pinia`, `axios`
- 配置 Vite 代理（`/api` → `http://localhost:8080`）
- 创建目录结构：`views/`, `router/`, `stores/`, `api/`
- 配置 Element Plus 全局引入
- 创建 `main.js` 入口文件，注册 Router、Pinia、Element Plus

**独立验证：**

```bash
cd frontend
npm run dev
```

浏览器访问 `http://localhost:5173`，应看到 Vite 默认页面或空白页面（无报错）。控制台无 `npm install` 遗漏的依赖错误。

```bash
# 验证构建

```

## 模块 2：Axios 封装与 API 层

**目标：** 统一封装 HTTP 请求，实现 Token 自动注入和 401 拦截。

- 创建 `api/request.js` — Axios 实例，baseURL 为 `/api`
- 请求拦截器：自动从 localStorage 读取 Token，写入 `Authorization` 头
- 响应拦截器：401 状态码自动跳转登录页
- 按业务拆分 API 模块：
  - `api/auth.js` — 登录、注册
  - `api/user.js` — 用户 CRUD
  - `api/content.js` — 帖子 CRUD
  - `api/comment.js` — 评论操作
  - `api/notification.js` — 通知查询、标记已读

**独立验证：**

```bash
npm run dev
```

在浏览器控制台手动测试：

```javascript
// 验证 API 模块可导入（无语法错误）
// 在 main.js 或任意组件中临时添加：
import { login } from './api/auth'
import { getUsers } from './api/user'
import { getContents } from './api/content'
console.log('API modules loaded OK')
```

浏览器控制台应输出 `API modules loaded OK`，无 import 错误。

再验证代理是否生效：

```bash
curl http://localhost:5173/api/auth/login -X POST -H "Content-Type: application/json" -d '{"username":"test","password":"test"}'
```

应返回 Gateway 转发的响应（非 404 或连接拒绝），说明 Vite 代理配置正确。

## 模块 3：路由与状态管理

**目标：** 配置前端路由和全局用户状态。

- 创建 `stores/user.js`（Pinia）：
  - State：`token`, `userId`, `username`, `name`
  - Actions：`login()`, `logout()`, `loadFromStorage()`
  - 登录成功后持久化到 localStorage
- 创建 `router/index.js`：
  - `/login` → Login.vue（占位组件即可）
  - `/register` → Register.vue（占位组件即可）
  - `/` → PostList.vue（占位组件即可）
  - `/post/new` → PostForm.vue（占位组件即可）
  - `/post/:id` → PostDetail.vue（占位组件即可）
  - `/post/:id/edit` → PostForm.vue（占位组件即可）
  - `/users` → UserManage.vue（占位组件即可）
  - `/notifications` → Notification.vue（占位组件即可）
- 路由守卫：未登录访问受保护页面时跳转 `/login`

**独立验证：**

为每个路由创建占位组件（仅含 `<template><div>XXX Page</div></template>`），然后：

```bash
npm run dev
```

浏览器逐一访问以下路径，确认页面渲染正确且无控制台报错：

| 访问路径 | 预期结果 |
|----------|----------|
| `http://localhost:5173/login` | 显示 "Login Page" |
| `http://localhost:5173/register` | 显示 "Register Page" |
| `http://localhost:5173/` | 应被路由守卫重定向到 `/login`（因无 Token） |
| `http://localhost:5173/users` | 应被重定向到 `/login` |

手动在浏览器控制台设置 Token 测试守卫放行：

```javascript
localStorage.setItem('token', 'fake-token')
// 然后访问 http://localhost:5173/
// 应显示 "PostList Page" 而非跳转登录
```

## 模块 4：认证页面（登录 + 注册）

**目标：** 实现用户登录和注册功能。

- `Login.vue`：
  - 用户名 + 密码输入框 + 登录按钮
  - 调用 `POST /api/auth/login`，成功后存 Token 到 Pinia + localStorage
  - 底部"没有账号？去注册"链接跳转注册页
- `Register.vue`：
  - 用户名 + 密码 + 确认密码 + 邮箱 + 注册按钮
  - 调用 `POST /api/auth/register`，成功后跳转登录页
- 使用 Element Plus 的 `ElForm` 表单验证

**独立验证：**

前提：后端 user-service 和 Gateway 已启动。

```bash
npm run dev
```

浏览器操作：

1. 访问 `http://localhost:5173/register`
2. 填写：用户名 `testuser`、密码 `123456`、确认密码 `123456`、邮箱 `test@example.com`
3. 点击注册 → 应提示成功并跳转到登录页
4. 输入 `testuser` / `123456` 点击登录 → 应提示成功并跳转到首页 `/`
5. 检查 `localStorage` 中应有 `token` 值
6. 刷新页面 → 应保持登录状态（不跳回登录页）

异常验证：
- 不填用户名直接点登录 → 应显示表单验证提示
- 输入错误密码 → 应显示后端返回的错误消息

## 模块 5：帖子列表（首页）

**目标：** 展示帖子卡片列表，支持分页和导航。

- `PostList.vue`：
  - 顶部导航栏：Logo + "发帖"按钮 + 通知铃铛（未读数 badge）+ 当前用户名 + 退出按钮
  - 帖子卡片列表：标题、作者名、发布时间、内容摘要
  - Element Plus `ElPagination` 分页组件
  - 点击卡片 → 帖子详情页
  - 点击作者名 → 该用户的帖子列表
- 调用 `GET /api/content` 获取帖子列表
- 调用 `GET /api/users` 获取用户信息用于显示作者名

**独立验证：**

前提：后端 user-service + content-service + Gateway 已启动，数据库中至少有 1 个用户和 1 篇帖子（通过 curl 或模块 4 注册的用户）。

```bash
# 先通过 API 创建测试数据
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"123456","email":"alice@example.com"}'

TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

curl -X POST http://localhost:8080/api/content \
  -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" \
  -d '{"userId":1,"title":"测试帖子","body":"这是测试内容"}'
```

然后浏览器操作：

1. 登录后访问首页 `/`
2. 应看到帖子卡片，显示标题、作者名、时间
3. 点击帖子卡片 → 跳转到帖子详情页（模块 7 实现后验证）
4. 点击"发帖"按钮 → 跳转到发帖页（模块 6 实现后验证）
5. 点击"退出" → 清除 Token 并跳转登录页

## 模块 6：发帖与编辑

**目标：** 实现帖子的创建和编辑功能。

- `PostForm.vue`：
  - 标题输入框 + 正文文本域 + 作者下拉选择 + 提交按钮
  - 新建模式：调用 `POST /api/content`
  - 编辑模式：根据路由参数 `id` 加载已有数据，调用 `PUT /api/content/{id}`
  - 提交成功后跳转帖子详情或列表页
- 使用 Element Plus `ElForm` 表单组件

**独立验证：**

前提：模块 4、5 已完成，用户已登录。

浏览器操作：

1. 首页点击"发帖"按钮 → 进入 `/post/new`
2. 填写标题和正文，选择作者，点击提交
3. 应提示发布成功并跳转到帖子详情页或首页
4. 回到首页，确认新帖子出现在列表中
5. 点击帖子进入详情页，点击"编辑" → 进入 `/post/:id/edit`
6. 修改标题，点击提交 → 应更新成功
7. 回到首页确认标题已更新

异常验证：
- 标题为空提交 → 应显示表单验证提示

## 模块 7：帖子详情与评论

**目标：** 展示帖子完整内容，支持评论的查看、发表和删除。

- `PostDetail.vue`：
  - 帖子内容展示区：标题、正文、作者、发布时间
  - 评论列表：评论者名、内容、时间、删除按钮
  - 发表评论区：评论者下拉选择 + 评论内容输入 + 提交按钮
- 调用 `GET /api/content/{id}` 获取帖子详情
- 调用 `GET /api/comments/content/{contentId}` 获取评论列表
- 调用 `POST /api/comments` 发表评论
- 调用 `DELETE /api/comments/{id}` 删除评论

**独立验证：**

前提：模块 5、6 已完成，帖子已创建。

浏览器操作：

1. 从首页点击帖子卡片 → 进入 `/post/:id`
2. 应显示帖子标题、正文、作者名、发布时间
3. 评论区应显示"暂无评论"或已有评论列表
4. 选择评论者，输入评论内容，点击提交 → 评论出现在列表中
5. 刷新页面 → 评论仍存在（从后端加载）
6. 点击评论的删除按钮 → 评论从列表中移除

异常验证：
- 评论内容为空提交 → 应显示验证提示
- 删除别人的评论（如后端有权限校验）→ 应显示错误提示

## 模块 9：通知中心

**目标：** 展示用户通知列表，支持标记已读。

- `Notification.vue`：
  - 用户切换下拉（用于演示不同用户的通知）
  - 通知列表：类型图标、内容、时间、已读/未读状态标签
  - 点击通知 → 调用标记已读接口
- 调用 `GET /api/notifications/user/{userId}` 获取通知列表
- 调用 `PUT /api/notifications/{id}/read` 标记已读

**独立验证：**

前提：content-service + Gateway 已启动，Kafka 运行中。

先创建通知数据：

```bash
# 登录获取 Token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 发一条评论（触发 Kafka 事件 → content-service 通知模块消费 → 生成通知）
curl -X POST http://localhost:8080/api/comments \
  -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" \
  -d '{"contentId":1,"userId":1,"body":"触发通知的评论"}'
```

浏览器操作：

1. 导航到 `/notifications`
2. 应看到通知列表，包含"您的帖子收到了新评论"类型的通知
3. 未读通知显示"未读"标签（红色）
4. 点击通知 → 调用标记已读接口 → 标签变为"已读"（绿色）
5. 刷新页面 → 已读状态保持
