# MySpringCloudDemo 微服务系统需求文档

## 1. 项目概述

### 1.1 项目背景

当前项目已完成 Maven 多模块骨架搭建，正在从单体 Spring Boot 应用逐步重构为 Spring Cloud 微服务架构。

### 1.2 项目目标

构建社交/内容平台微服务系统，实现：
- 服务注册与发现（Nacos）
- 统一 API 网关 + JWT 鉴权（Spring Cloud Gateway）
- 服务间同步调用（OpenFeign）
- 异步事件驱动（Kafka）
- 熔断降级（Sentinel/Resilience4j）
- 分布式链路追踪（Micrometer + Zipkin）
- 前后端分离（Vue 3 + Element Plus）

### 1.3 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| JDK | 21 | 运行环境 |

### 2.2 服务间通信

| 调用方 | 被调用方 | 方式 | 场景 |
|--------|----------|------|------|
| content-service | user-service | OpenFeign (同步) | 创建帖子时验证用户是否存在 |
| content-service (评论) | content-service (帖子) | 内部调用 | 创建评论时验证帖子存在 |
| content-service (评论) | content-service (通知) | Kafka (异步) | 评论新增事件 → 生成通知 |

## 4. 功能需求

### 4.1 公共模块 (common-module)

#### 4.1.1 统一异常处理
- `BusinessException` — 业务异常基类（含错误码）
- `ResourceNotFoundException` — 资源未找到异常
- `GlobalExceptionHandler` — 全局异常处理器

#### 4.1.2 统一响应封装
- `ApiResponse<T>` 统一响应结构：
  ```json
  { "code": 200, "message": "success", "data": { ... } }
  ```

#### 4.1.3 JWT 工具
- `JwtUtil.generateToken(userId, username)` — 签发 Token（24h 有效）
- `JwtUtil.parseToken(token)` — 解析 Token
- `JwtUtil.getUserIdFromToken(token)` / `getUsernameFromToken(token)`

### 4.2 网关服务 (gateway-service)

#### 4.2.1 路由规则
| 路径前缀 | 目标服务 | 说明 |
|----------|----------|------|
| /api/auth/** | user-service | 认证接口（白名单，不校验 Token） |
| /api/users/** | user-service | 用户管理（需 Token） |
| /api/content/** | content-service | 帖子管理（需 Token） |
| /api/comments/** | content-service | 评论管理（需 Token） |
| /api/notifications/** | content-service | 通知管理（需 Token） |

#### 4.2.2 JWT 鉴权
- 白名单路径（`/api/auth/register`, `/api/auth/login`）直接放行
- 其他路径校验 `Authorization: Bearer <token>` 头
- 校验通过 → 解析 `userId`、`username` 写入请求头转发给下游
- 校验失败 → 返回 `401 Unauthorized`

### 4.3 用户服务 (user-service)

#### 4.3.1 接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 用户注册 |
| POST | /api/auth/login | 用户登录，返回 JWT Token |
| GET | /api/auth/me | 获取当前用户信息（需 Token） |
| GET | /api/users | 查询所有用户 |
| GET | /api/users/{id} | 根据 ID 查询 |
| PUT | /api/users/{id} | 更新用户 |
| DELETE | /api/users/{id} | 删除用户 |

#### 4.3.2 数据库表
```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,  -- BCrypt 加密
    name VARCHAR(50),
    email VARCHAR(100),
    phone VARCHAR(20)
);
```

### 4.4 内容服务 (content-service)

内容服务整合了帖子管理、评论管理和通知管理三大功能，内部通过 Kafka 实现评论→通知的异步事件驱动。

#### 4.4.1 帖子管理接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/content | 发布帖子 |
| GET | /api/content | 查询帖子列表 |
| GET | /api/content/{id} | 查询单篇帖子 |
| GET | /api/content/user/{userId} | 查询某用户的帖子 |
| PUT | /api/content/{id} | 更新帖子 |
| DELETE | /api/content/{id} | 删除帖子（软删除） |

#### 4.4.2 评论管理接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/comments | 新增评论 |
| GET | /api/comments/content/{contentId} | 查询某帖子的评论 |
| DELETE | /api/comments/{id} | 删除评论 |

#### 4.4.3 通知管理接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/notifications/user/{userId} | 查询用户的通知 |
| PUT | /api/notifications/{id}/read | 标记已读 |

#### 4.4.4 数据库表
```sql
CREATE TABLE contents (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    body TEXT,
    status VARCHAR(20) DEFAULT 'PUBLISHED',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE comments (
    id BIGSERIAL PRIMARY KEY,
    content_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    body TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE notifications (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    type VARCHAR(50) NOT NULL,
    message TEXT NOT NULL,
    read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.4.5 Kafka 事件
- 新增评论时发送 `comment-events` Topic（含 authorId）
- 内部消费 `comment-events` → 为帖子作者生成通知

## 5. 接口调用链示例

### 5.1 用户注册 → 登录 → 发帖
```
Client → POST /api/auth/register → user-service → 返回用户
Client → POST /api/auth/login    → user-service → 返回 JWT Token
Client → POST /api/content (带 Token)
       → Gateway (JWT 鉴权通过)
       → content-service
         → Feign: user-service GET /api/users/{id} (验证用户)
         → 写入 contents 表
         → Kafka: content-events
       → 返回帖子
```

### 5.2 评论帖子 → 触发通知
```
Client → POST /api/comments (带 Token)
       → Gateway → content-service (评论模块)
         → ContentService.findById() (内部调用，验证帖子)
         → 写入 comments 表
         → Kafka: comment-events
       → 返回评论

content-service (通知模块) ← Kafka 消费 comment-events
                           → 写入 notifications 表
```

## 7. 验收标准

### 7.1 编译
- `mvn clean package -DskipTests` 全部模块编译通过

### 7.2 服务注册
- 3 个服务在 Nacos 控制台可见（gateway-service、user-service、content-service）

### 7.3 接口功能
- 通过 Gateway (8080) 可正常访问所有服务的 API
- JWT 鉴权：无 Token 返回 401，有 Token 正常转发
- OpenFeign 调用链路：content-service → user-service
- Kafka 事件：content-service 内部评论模块 → 通知模块

### 7.4 异常处理
- 查询不存在的资源返回 404 + 统一错误格式
- 业务异常返回 400 + 统一错误格式
